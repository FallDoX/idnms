# GSD (Get Shit Done) Workflow

Когда пользователь вводит `/gsd:help` или `/gsd:validate` и другие команды, используй этот workflow.

## Доступные команды

| Команда | Действие |
|---------|----------|
| `/gsd:help` | Показать справку по GSD командам |
| `/gsd:status` | Показать текущий статус проекта |
| `/gsd:validate` | Выполнить валидацию кода |
| `/gsd:debug` | Активировать режим отладки |
| `/gsd:research` | Исследовательский режим |
| `/gsd:todo` | Показать/обновить список задач |
| `/gsd:uat` | Пользовательское тестирование |
| `/gsd:milestone` | Работа с вехами проекта |
| `/gsd:roadmap` | Дорожная карта проекта |
| `/gsd:state` | Текущее состояние проекта |
| `/gsd:summary` | Сводка по проекту |
| `/gsd:project` | Информация о проекте |
| `/gsd:requirements` | Требования проекта |
| `/gsd:verify-work` | Проверка выполненной работы |
| `/gsd:plan` | Планирование этапов |
| `/gsd:discover` | Фаза обнаружения/исследования |
| `/gsd:context` | Контекст проекта |
| `/gsd:continue` | Продолжить работу |
| `/gsd:ui-spec` | Спецификация UI |

## Инструкции

1. **Для `/gsd:help`:**
   - Покажи список доступных GSD команд
   - Укажи их назначение и примеры использования

2. **Для `/gsd:validate`:**
   - Проверь синтаксис всех TS/TSX файлов
   - Запусти `npm run build` для проверки компиляции
   - Проверь наличие необходимых компонентов
   - Отчитайся о результатах валидации

3. **Для `/gsd:status`:**
   - Проверь todo_list текущего состояния
   - Покажи прогресс по задачам
   - Укажи, что осталось сделать

4. **Для `/gsd:debug`:**
   - Активируй детальное логирование
   - Проверь консольные ошибки
   - Предложи точки для дебага

5. **Для `/gsd:todo`:**
   - Используй todo_list tool для показа текущих задач
   - Предложи обновить приоритеты при необходимости

6. **Для `/gsd:uat`:**
   - Запусти режим пользовательского тестирования
   - Создай/обнови файл UAT.md с тестами

7. **Для `/gsd:milestone`:**
   - Покажи текущие вехи проекта
   - Обнови MILESTONE.md при достижении вехи

8. **Для `/gsd:roadmap`:**
   - Покажи дорожную карту проекта
   - Используй шаблон из ROADMAP.md

9. **Для `/gsd:state`:**
   - Покажи текущее состояние проекта
   - Фаза, план, прогресс

10. **Для `/gsd:summary`:**
    - Создай сводку по проекту
    - Краткий обзор прогресса

11. **Для `/gsd:project`:**
    - Покажи информацию о проекте
    - Цели, ограничения, соглашения

12. **Для `/gsd:requirements`:**
    - Покажи требования проекта
    - Список REQ-XX

13. **Для `/gsd:verify-work`:**
    - Проверь выполненную работу
    - Запусти полный набор тестов

14. **Для `/gsd:plan`:**
    - Перейди в режим планирования этапов
    - Создай/обнови план фазы

15. **Для `/gsd:discover`:**
    - Фаза обнаружения требований
    - Исследуй codebase и пользовательские нужды

16. **Для `/gsd:context`:**
    - Покажи контекст проекта
    - Решения, предпочтения пользователя

17. **Для `/gsd:continue`:**
    - Продолжи работу с последнего состояния
    - Загрузи STATE.md или continue-here.md

18. **Для `/gsd:ui-spec`:**
    - Покажи/создай спецификацию UI
    - Используй шаблон UI-SPEC.md

## Структура проекта GSD

- `.claude/get-shit-done/templates/` — шаблоны для разных фаз:
  - DEBUG.md — отладка
  - UAT.md — пользовательское тестирование
  - VALIDATION.md — валидация
  - state.md — состояние проекта
  - project.md — информация о проекте
  - requirements.md — требования
  - research.md — исследование
  - context.md — контекст
- `.claude/get-shit-done/commands/` — команды GSD
