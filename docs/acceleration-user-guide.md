# Руководство по анализу ускорения

## Введение

Функция анализа ускорения позволяет автоматически определять попытки разгона в данных телеметрии и анализировать их эффективность. Это помогает оценить производительность вашего электровелосипеда в различных условиях.

## Загрузка и анализ данных

### 1. Загрузка CSV файла

Перетащите CSV файл в окно приложения или нажмите кнопку "Upload Trip CSV". Приложение автоматически определит формат файла (старый или новый) и загрузит данные.

### 2. Автоматическое определение ускорений

После загрузки данных приложение автоматически:
- Определяет попытки разгона на основе настроенных порогов
- Рассчитывает метрики для каждой попытки
- Отображает результаты в таблице

## Конфигурация порогов

### Настройка диапазонов скорости

1. Откройте секцию "Конфигурация ускорения"
2. Выберите пресет или настройте кастомные пороги:
   - **0-60 км/ч** - Стандартный тест разгона
   - **0-90 км/ч** - Для высокоскоростных поездок
   - **0-100 км/ч** - Максимальный разгон
   - **Кастомный** - Настройте свои значения

3. Приложение автоматически пересчитает попытки при изменении порогов

### Пресеты

- **Стандарт** - 0-60 км/ч (рекомендуется для городской езды)
- **Высокий** - 0-90 км/ч (для скоростных трасс)
- **Максимальный** - 0-100 км/ч (для максимальной производительности)

## Использование таблицы попыток

### Основные метрики

Таблица показывает следующие метрики для каждой попытки:

- **№** - Порядковый номер попытки
- **Диапазон** - Пороги скорости (например, 0-60)
- **Время (с)** - Время разгона в секундах
- **Дистанция (м)** - Пройденное расстояние в метрах
- **Пик. мощн. (Вт)** - Максимальная мощность
- **Ср. мощн. (Вт)** - Средняя мощность
- **Пад. бат. (%)** - Падение заряда батареи
- **Ср. темп. (°C)** - Средняя температура

### Расширенные метрики

Дополнительные метрики для глубокого анализа:

- **Эфф. мощн.** - Мощность на единицу прироста скорости (Вт/(км/ч))
- **Согл. мощн.** - Стандартное отклонение мощности (0-1)
- **Пад. бат./с** - Скорость разряда (%/с)
- **Энерг./км** - Расход энергии (Wh/km)
- **Корр. темп.** - Корреляция между температурой и мощностью
- **Эфф. темп.** - Эффективность по температуре (оптимальный диапазон 20-35°C)

### Фильтрация и сортировка

**Фильтрация:**
- Нажмите кнопку "Неполные" для отображения незавершённых попыток
- Используйте фильтры по мощности и температуре
- Сбросьте фильтры кнопкой "Сбросить"

**Сортировка:**
- Нажмите на заголовок столбца для сортировки
- Повторное нажатие меняет направление сортировки
- Сортировка доступна для всех метрик

### Выбор колонок

1. Нажмите кнопку "Отображаемые столбцы"
2. Отметьте метрики, которые хотите видеть
3. Кнопка "Основные" сбрасывает выбор на важные метрики

## Режим сравнения

### Выбор попыток для сравнения

1. Откройте секцию "Сравнение"
2. В таблице попыток отметьте чекбоксы рядом с нужными попытками
3. Секция сравнения автоматически обновится

### Наложенные графики

Графики отображают:
- Кривые скорости всех выбранных попыток на одном графике
- Цветовое кодирование для каждой попытки
- Лучшая попытка выделена более толстой линией
- Опционально: кривая мощности на отдельной оси

### Фильтрация в режиме сравнения

- **Все** - Показать все выбранные попытки
- **Лучшие N** - Показать N самых быстрых попыток (настраиваемый лимит)
- **Худшие N** - Показать N самых медленных попыток

Настройте лимит фильтра с помощью поля ввода рядом с кнопками.

### Таблица дельта-метрик

Таблица показывает разницу каждой попытки относительно лучшей:
- **Δ Время** - Разница во времени (секунды)
- **Δ Дистанция** - Разница в дистанции (метры)
- **Δ Мощность** - Разница в мощности (ватты)
- **Δ Батарея** - Разница в падении батареи (%)

Положительные значения означают, что метрика хуже, чем у лучшей попытки.

### Кривая мощности

Включите кнопку "Мощность" для отображения:
- Кривых мощности на отдельной оси справа
- Сравнение мощности между попытками
- Анализ эффективности разгона по мощности

## Понимание метрик

### Эффективность разгона

**Время разгона** - Основной показатель производительности. Чем меньше время, тем лучше разгон.

**Дистанция** - Расстояние, пройденное за время разгона. Оптимальный баланс между временем и дистанцией.

**Мощность** - Мощность влияет на ускорение. Более высокая мощность обычно означает более быстрый разгон.

**Батарея** - Падение заряда показывает энергоэффективность. Меньшее падение при том же разгоне - лучше.

### Температура

Оптимальный диапазон температуры: 20-35°C
- Ниже 20°C - снижается эффективность батареи
- Выше 35°C - возможна деградация компонентов

**Эффективность температуры** - Нормализованный показатель (0-1), где 1 - оптимальная температура.

### Расширенные метрики

**Эффективность мощности** - Мощность на единицу прироста скорости. Низкие значения указывают на неэффективное использование мощности.

**Согласованность мощности** - Стандартное отклонение мощности. Высокие значения (ближе к 1) означают стабильную подачу мощности.

**Распределение мощности** - Процент времени в диапазонах:
- Низкая (<1000W) - экономичная езда
- Средняя (1000-2000W) - нормальный разгон
- Высокая (>2000W) - максимальная производительность

## Советы и рекомендации

### Для улучшения результатов

1. **Выберите правильные пороги** - Настройте пороги в соответствии с вашими условиями езды
2. **Фильтруйте данные** - Используйте фильтры для исключения аномальных попыток
3. **Сравнивайте похожие условия** - Сравнивайте попытки в похожих условиях (температура, заряд батареи)
4. **Анализируйте тренды** - Используйте расширенные метрики для поиска паттернов

### Интерпретация результатов

- **Хороший разгон**: Время < 10с для 0-60 км/ч, стабильная мощность, умеренное падение батареи
- **Проблемы с мощностью**: Низкая эффективность мощности, нестабильная подача
- **Проблемы с температурой**: Высокая температура при разгоне, низкая эффективность температуры

### Оптимизация

1. **Предварительный разогрев** - Разогните батарею перед измерениями (несколько минут езды)
2. **Полный заряд** - Измеряйте при полном заряде для сравнимых результатов
3. **Постоянные условия** - Измеряйте в похожих условиях для трендов
4. **Калибровка** - Регулярно калибруйте датчики скорости

## Устранение неполадок

### Нет попыток разгона

- Проверьте, что пороги настроены корректно
- Убедитесь, что данные содержат достаточный диапазон скоростей
- Попробуйте включить "Неполные попытки"
- Проверьте формат CSV файла

### Странные значения метрик

- Убедитесь, что данные корректны (нет пропусков, аномалий)
- Проверьте калибровку датчиков
- Фильтруйте попытки с экстремальными значениями

### Графики не отображаются

- Убедитесь, что выбраны попытки для сравнения
- Проверьте, что данные содержат временные метки
- Попробуйте сбросить фильтры
