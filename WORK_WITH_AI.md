# Как работать с AI над проектом

## Основные принципы

### 1. Один чат = одна задача
**НЕ** смешивайте разные задачи в одном чате. Создавайте новый чат для каждой крупной задачи.

**Примеры правильного подхода:**
- Чат 1: "Добавь кастомные пресеты в AccelerationTab"
- Чат 2: "Исправь мобильную адаптивность кнопок"
- Чат 3: "Добавь графики для GPSMap"

**Пример неправильного подхода:**
- Чат 1: "Добавь пресеты, исправь адаптивность, добавь графики, почини баги..."

### 2. Маленькие атомарные изменения
Просите AI делать маленькие изменения за раз. Это легче откатить и протестировать.

**Плохо:**
```
"Перепиши весь компонент AccelerationTab с нуля"
```

**Хорошо:**
```
"Добавь кнопку для создания кастомного пресета"
"Измени логику фильтрации попыток"
"Добавь текст с названиями пресетов на кнопки"
```

### 3. Проверяйте каждое изменение
После каждого изменения:
1. Посмотрите diff в IDE
2. Запустите браузер (@br)
3. Проверьте функционал
4. Только потом просите следующее изменение

### 4. Используйте Git правильно
- **Маленькие коммиты** - один функционал = один коммит
- **Описательные сообщения** - `feat: Add custom presets` или `fix: Fix infinite loop`
- **Не откатывайтесь назад без понимания** - используйте `git log --oneline` чтобы понять историю

### 5. Просите объяснения перед реализацией
Если задача сложная, попросите AI сначала объяснить план:
```
"Объясни как ты будешь решать эту задачу"
"Какие файлы нужно будет изменить?"
"Какие риски?"
```

## Шаблоны запросов

### Для новой функции
```
"Добавь [функция] в [компонент]. 
Сначала объясни план, потом реализуй."
```

### Для исправления бага
```
"Исправь [описание бага]. 
Сначала проанализируй причину, потом предложи решение."
```

### Для рефакторинга
```
"Рефактори [код] для [цель]. 
Не меняй логику, только структуру."
```

### Для вопросов
```
"Объясни как работает [функция/компонент]"
```

## Команды для работы с AI

### @br - Запустить браузер
Запускает dev сервер и открывает браузер для проверки изменений.

### @file - Ссылка на файл
Используйте для указания конкретного файла:
```
@[/path/to/file.tsx] добавь кнопку
```

### @element - Ссылка на DOM элемент
Используйте для указания конкретного элемента в браузере:
```
@[dom-element:button:AccelerationTab] исправь стиль
```

## Git команды для работы с AI

### Посмотреть историю
```bash
git log --oneline -10
```

### Откат на 1 коммит
```bash
git reset --hard HEAD~1
git push -f
```

### Откат на конкретный коммит
```bash
git reset --hard <commit-hash>
git push -f
```

### Вернуться к origin/main
```bash
git reset --hard origin/main
git push -f
```

### Создать новую ветку
```bash
git checkout -b feature/my-feature
```

## Типичные ошибки и как их избежать

### Ошибка 1: Слишком большие изменения
**Проблема:** AI меняет сразу много файлов, сложно откатить
**Решение:** Разбейте задачу на маленькие шаги

### Ошибка 2: Не проверяете diff
**Проблема:** AI удалил важный код, вы не заметили
**Решение:** Всегда смотрите diff в IDE перед коммитом

### Ошибка 3: Откатываетесь назад без понимания
**Проблема:** `git reset --hard HEAD~5` удалило 5 коммитов работы
**Решение:** Используйте `git log --oneline -10` чтобы понять что удаляете

### Ошибка 4: Смешиваете задачи в одном чате
**Проблема:** Контекст переполнен, AI теряет нить
**Решение:** Один чат = одна задача

### Ошибка 5: Не тестируете после изменений
**Проблема:** Баги накапливаются
**Решение:** После каждого изменения запустите @br и проверьте

## Как правильно работать над функциями

### Шаг 1: Планирование
```
"Мне нужно добавить [функция]. Объясни план реализации."
```

### Шаг 2: Реализация
```
"Реализуй первый шаг плана."
```

### Шаг 3: Проверка
```
@br
"Проверь что [функция] работает."
```

### Шаг 4: Коммит
```bash
git add .
git commit -m "feat: Add [function]"
git push
```

### Шаг 5: Следующий шаг
```
"Реализуй следующий шаг."
```

## Пример правильного workflow

### Чат 1: Добавить кастомные пресеты
```
User: "Добавь возможность создавать кастомные пресеты в AccelerationTab. Сначала объясни план."
AI: "План: 1) Добавить state для кастомных пресетов, 2) Добавить форму создания, 3) Обновить логику фильтрации, 4) Добавить UI для отображения кастомных пресетов"
User: "Реализуй шаг 1"
AI: [изменения]
User: @br
User: "Работает. Реализуй шаг 2"
...
```

### Чат 2: Исправить мобильную адаптивность
```
User: "Исправь мобильную адаптивность кнопок в AccelerationTab. Сначала проанализируй проблему."
AI: "Проблема: кнопки слишком маленькие для тач-интерфейса. Решение: увеличить min-height до 44px"
User: "Реализуй"
AI: [изменения]
User: @br
User: "Работает. Коммити."
```

## Как просить AI не ломать вещи

### Будьте конкретны
```
"Добавь кнопку НЕ удаляя существующий код"
"Измени логику фильтрации НЕ меняя UI"
"Рефактори компонент НЕ меняя поведение"
```

### Просите объяснения
```
"Что именно ты будешь менять?"
"Какие файлы затронуты?"
"Есть ли риски?"
```

### Проверяйте diff
Всегда смотрите diff в IDE перед коммитом.

## Когда начинать новый чат

Начинайте новый чат когда:
- Вы закончили текущую задачу и закоммитили
- Задача существенно отличается от предыдущей
- Контекст чата стал слишком большим
- AI начинает терять нить разговора

## Резюме

1. **Один чат = одна задача**
2. **Маленькие изменения**
3. **Проверяйте каждое изменение**
4. **Используйте Git правильно**
5. **Просите объяснения**
6. **Смотрите diff**
7. **Тестируйте в браузере (@br)**

Следуйте этим принципам и работа с AI станет эффективной и безопасной.
